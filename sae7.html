Développer un composant d'une solution décisionnelle

Développer un composant d'une solution décisionnelle consiste à concevoir une brique logicielle qui contribue à la collecte, au traitement, ou à l’analyse des données afin d’aider à la prise de décision. Ces composants peuvent inclure des interfaces d’acquisition de données, des algorithmes d’analyse, des visualisations, ou encore des systèmes d’automatisation comme des API ou des scripts intelligents. L’objectif est de transformer des données brutes en informations exploitables, le tout dans un cadre automatisé, structuré et réutilisable.

Dans le cadre de cette SAE appelé: Développement d'un composant d'une solution décisionelle, j’ai développé un script Python appelé app.py, utilisant le framework Flask. Ce composant a pour but de faciliter le classement automatique de transactions financières.

Ce projet propose une interface web simple permettant à un utilisateur d’uploader un fichier de transactions (CSV, Excel ou JSON).

Le script analyse ensuite la colonne description de chaque transaction et attribue une catégorie prédéfinie (loyer, salaire, alimentation, loisirs, etc.) en fonction de mots-clés. Les résultats sont retournés au format JSON.

Fonctionnalités clés

  • Validation du format et des colonnes requises.
  • Traitement intelligent avec pandas.
  • Retour JSON des résultats catégorisés.
  • Support de requêtes curl pour automatiser les envois.
  • Suppression automatique des fichiers temporaires après analyse.

Ce composant peut être intégré dans un processus décisionnel financier pour aider à mieux comprendre les postes de dépenses ou de revenus de manière automatisée.

Ce projet m’a aidé à mieux comprendre comment un outil informatique peut aider à prendre des décisions. Le script que j’ai créé permet d’automatiser une tâche longue et répétitive, comme le classement des dépenses, de façon rapide et fiable. Il fonctionne bien, mais il pourrait encore être amélioré : par exemple, en utilisant l’intelligence artificielle pour mieux reconnaître les catégories, en ajoutant des graphiques pour visualiser les résultats, ou en enregistrant les données dans une base de données pour garder un historique. Même si c’est un projet simple, il montre comment un petit programme peut devenir un vrai outil utile dans une solution d’aide à la décision.
